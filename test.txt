/// generated migrations target PostgreSQL, independent of the runtime
/// environment or connection string. It is never used at runtime.
///
/// The connection string here is a placeholder — `migrations add` only needs a
/// provider to scaffold SQL and does NOT connect to a database. `database update`
/// would connect, so pass a real one via ACPP_DB_CONNECTION or run the app
/// (which applies migrations automatically on Postgres).
/// </summary>
